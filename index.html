Monterey Bay AI Lab

We train evidence engines across water, biology, and field-risk questions, then show the scorecard.

Mission Control is the public notebook for the whole project: pulling water, weather, ocean, watershed, biology, toxin, fisheries, and sensor records into one lakehouse; using frontier models to scout new engines and failure checks; running the real jobs on local GPU compute; then publishing what beat a fair baseline, what only partly worked, and what failed.

What we are doing

Building a repeatable test bench for multi-hazard science and operations.

Each run starts with a concrete question: can we rank bacteria risk, toxic HAB onset, cyanotoxin risk, hypoxia, storm response, fisheries effort, wildlife risk, or cross-region transfer better than recent same-place history, persistence, or seasonality? The point is not to make every model look good. The point is to find the few places where extra modeling is worth it.

How we work

Train a lot, promote little, keep the misses visible.

The engines compete against boring but strong baselines first. A result does not become a claim until it has a held-out read, a baseline comparison, and a failure mode written down. That is why the site shows wins, caveats, and nulls side by side instead of only the best-looking chart.

The pipeline

Raw records land in the lakehouse, transforms make them comparable, training engines compete against baseline rules, and evidence gates decide what can be shown as a finding.

How to read this site

Result grades

Supported
Held up under audit — beat a real baseline on held-out data, with confidence intervals.
Caveated
A real signal, but with limits you should read (narrow slice, soft generalization, screening-only).
Null
Tested and did not clear the bar. Not hidden — kept visible so it isn't re-tried blindly.
Exploratory
An open lane; correlations exist but haven't survived strict cross-validation yet.

Signal verdicts (what a feature did for a target)

PRIMARY / KEEP
Added real, audited skill over the baseline.
WASH
No lift once the baseline was fair.
REJECT
Hurt the model.
UNTESTED
Not yet gated.

The numbers

AUC
Ranking skill: 0.5 ≈ random, 1.0 = perfect ordering of event vs non-event days.
AP
Average precision — for rare events, how high real events rank. ΔAP is the gain over a baseline.
Baseline
The honest comparator a model must beat — usually persistence (yesterday) or seasonal climatology, not a strawman.
Held-out / perm-null
Test rows kept out of fitting; a permutation null checks the lift isn't luck.

Discovery Engines

This is the search machinery behind the results. We use frontier models to find candidate engines, translate them into runnable jobs, execute the real experiments on the RTX 5090 workstation, and only promote outputs that beat a fair baseline on held-out data.

01 Scout

Use frontier models to widen the search.

Frontier AI helps review papers, propose model families, spot missing controls, and draft job specs. It is a scout, not the judge.

02 Build

Turn ideas into reproducible jobs.

Each candidate becomes a script with a target, split, baseline, metric, artifact path, and confusion-matrix status when labels allow it.

03 Run

Execute on local compute.

GPU-heavy jobs run on the RTX 5090 box; CPU screens run as cheaper sweeps first. The output is saved before anyone writes a claim.

04 Gate

Promote, scope, or kill.

A result has to beat the right plain baseline, survive leakage checks, and pass critic review. Otherwise it stays a null or a data ask.

Glossary

This lab is careful about words. An untrained XGBoost or neural net is a learner, not a model — it only becomes a trained model once it's fitted, evaluated, and registered. A baseline is an honest comparator, not a discovery; a signal carries verdicts (KEEP / WASH / REJECT) per target. Each term below links to its real examples from the registry and signal catalog.

About this snapshot

A public science snapshot from the Monterey Bay AI Lab. Every result is graded honestly — Supported held up under audit, Caveated is real but limited, Null was tested and didn't hold — and nulls stay on the record beside the wins. The map shows observation coverage, never an extrapolated prediction surface. Evidence and access notes live under Data & reproduce.
